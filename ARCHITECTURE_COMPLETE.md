## 1. ОБЗОР АРХИТЕКТУРЫ

### 1.1 Принцип: многоуровневая система обработки информации

```
ОКРУЖЕНИЕ (мухи, свет, звуки)
         ↓
    СЕНСОРНАЯ СИСТЕМА (16-канальное зрение)
         ↓
    ОБРАБОТКА (Тектум, Ретина)
         ↓
    ЦЕНТРАЛЬНОЕ ПРИНЯТИЕ РЕШЕНИЙ (Мозг)
         ↓
    НЕЙРОМОДУЛЯЦИЯ (Дофамин, Серотонин, Глутамат)
         ↓
    ЗАДНЕМОЗГОВЫЕ СИСТЕМЫ (Метаболизм, Глия)
         ↓
    МОТОРНЫЙ ВЫХОД (Скорость тела, Язык)
         ↓
    ФИЗИКА (Pymunk, движение)
```

### 1.2 Критическая архитектурная особенность: ДВЕ СИСТЕМЫ ЭНЕРГИИ

Это главное: в BioFrog не одна энергия, а ДВЕ параллельные системы:

**СИСТЕМА 1: БИОЛОГИЧЕСКАЯ ЭНЕРГИЯ (0-1 шкала)**
- Источник: метаболизм (glucose_level в SystemicMetabolism)
- Назначение: модулирует поведение через глиальную сеть (глия видит энергию)
- Динамика: медленная, реалистичная
- Влияние: на дофамин, серотонин, возбудимость нейронов

**СИСТЕМА 2: ИГРОВАЯ МЕХАНИКА ЭНЕРГИИ (0-30 шкала, как ANN/SNN)**
- Источник: счетчик в BioFrogAgent (self.game_energy)
- Назначение: совместимость с ANN/SNN для сравнения
- Динамика: быстрая, детерминированная
- Влияние: прямо видна мозгу, модулирует сенсорное восприятие

## 2. ДЕТАЛЬНЫЕ КОМПОНЕНТЫ

### 2.1 СЕНСОРНАЯ СИСТЕМА (RetinalProcessing)

**Назначение:** Преобразовать сырые визуальные данные в нейрональный код

**Входы:**
- visual_scene: список кортежей (x, y, brightness) от мух в поле зрения

**Обработка:**
- Преобразование в 16-канальное представление (16 колонок в тектуме)
- Каждый канал кодирует направление и интенсивность света
- Блюринг соседних каналов (рецептивное поле)

**Выходы:**
- retinal_output: массив [16] с интенсивностями

**Формула кодирования:**
```
Для каждой мухи:
  - Вычислить направление от лягушки до мухи
  - Найти соответствующий канал (16 каналов = 360°/16 ≈ 22.5° на канал)
  - Интенсивность = brightness (зависит от расстояния до мухи)
  - Добавить шум для реалистичности
```

**Энергетическая модуляция (ВАЖНО):**
```python
energy_excitability_factor = 0.5 + 0.5 * game_energy_ratio  # Диапазон 0.5-1.0
retinal_output *= energy_excitability_factor
# Низкая энергия (0.0) → весь выход умножается на 0.5 (видит половину)
# Полная энергия (1.0) → выход без изменений
```

### 2.2 ТЕКТУМ (Tectum) - центр обработки

**Назначение:** Пространственная обработка визуальной информации, выбор цели

**Входы:**
- retinal_output: [16] от ретины
- motion_vectors: список движений мух

**Процесс:**
1. Для каждого канала возникает активность нейронов (50 нейронов на колонку × 16 = 800 нейронов)
2. Нейроны создают рецептивные поля - "карту" видимого пространства
3. Конкуренция между нейронами - "winner takes more" (не all-or-nothing)
4. Результат: 16-канальный выход (tectal_output)

**Вывод:**
- Направление: найти канал с максимальной активностью
- Интенсивность: использовать это значение для выбора цели
- movement_command: направление куда двигаться и куда бросить язык

---

### 2.3 СЕНСОРНАЯ ВНИМАТЕЛЬНОСТЬ (Attention Map)

**Назначение:** Селективное усиление важных сигналов

**Механизм:**
- Внимание фокусируется на мухе с самой высокой активностью в тектуме
- Рецептивное поле внимания: коническое расширение от центра внимания
- Усиление: сигналы внутри поля × 1.5, снаружи × 0.5

---

### 2.4 ГЛИАЛЬНАЯ СЕТЬ (GlialNetwork) - ЭНЕРГЕТИЧЕСКИЙ ИНТЕГРАТОР

**Критическое значение:** Глия - главная система для энергетической модуляции!

**架構:**
- 25 астроцитов (5×5 сетка)
- Каждый астроцит покрывает ~32 нейронов тектума
- Астроциты отпускают глиотрансмиттеры (модулируют синапсы)

**Входы:**
- tectal_activity: [16] активности тектума
- tectum_positions: позиции нейронов
- energy_level: биологическая энергия (0-1) 
- dt: временной шаг

**Процесс в каждом астроцыте:**
1. Считать среднюю активность нейронов вблизи
2. Вычислить метаболический спрос = активность × 0.1
3. **ЭНЕРГЕТИЧЕСКАЯ МОДУЛЯЦИЯ:**
   ```python
   energy_cost_factor = max(0.3, energy_level)  # 30-100% при низкой энергии
   sustainable_level = energy_cost_factor * 1.0  # Определить что может поддерживаться
   
   if metabolic_demand > sustainable_level:
       астроцит переходит в режим дефицита
       глиотрансмиттер_release *= 0.3  # Снижение выделения
       ослабление синапсов (мозг становится менее ответственным)
   else:
       нормальный режим работы
   ```

4. Выпуск глиотрансмиттеров (глутамата, ATP, GABA):
   ```python
   # Без дефицита энергии - полное выделение
   gliotransmitter_release = avg_activity * 0.5
   
   # С дефицитом энергии
   gliotransmitter_release *= energy_cost_factor  # Дополнительная модуляция
   ```

5. Модуляция синапсов:
   ```python
   synapse.weight *= (1.0 + gliotransmitter_release * 0.05)
   # Высокий глиотрансмиттер → усиление синапсов
   # Низкий → ослабление
   ```

**Выходы:**
- average_gliotransmitter: средний уровень глиотрансмиттера
- synapse_modulation: модуляция всех синапсов
- energy_stress_signal: есть ли энергетический стресс

### 2.5 НЕЙРОМОДУЛЯЦИЯ (Dopamine & Serotonin)

**Назначение:** Модулировать возбудимость и обучение в зависимости от статуса

**ДОФАМИН (Dopamine_level):**
- Иницыальное значение: 
  - Детство (juvenile): 0.85 (высокое исследование)
  - Взрослость: 0.5 (нормальный уровень)
- Награда за поимку мухи: +0.3 (дофаминовый всплеск)
- Энергетическое влияние:
  ```python
  dopamine *= max(0.3, energy_level)  # Низкая энергия → подавление дофамина
  # Это сообщает: "нет энергии, не исследуй, экономь"
  ```
- Эффект: увеличивает возбудимость нейронов, усиливает обучение

**СЕРОТОНИН (Serotonin_level):**
- Иницыальное значение:
  - Детство: 0.75
  - Взрослость: 0.5
- Сатиация (много мух поймана): -0.1 за каждую ловлю
- Восстановление: +0.01 за шаг (медленное)
- Энергетическое влияние:
  ```python
  serotonin *= max(0.5, energy_level * 1.0)  # Энергия влияет слабее чем на дофамин
  ```
- Эффект: модулирует настроение и готовность к поиску

**Эффект на мозг:**
```python
# В сенсорных нейронах
neuropeptide_concentration = dopamine_level + serotonin_level  # 0-1.6
# Это меняет threshold возбудимости (низкий threshold при высоком neuropeptide)
```

### 2.6 МЕТАБОЛИЗМ (SystemicMetabolism)

**Назначение:** Симулировать энергетические и химические процессы в теле

**Состояние:**
- glucose_level: уровень глюкозы (0-1, нормально ≈ 0.7)
- lactate_level: молочная кислота от анаэробного метаболизма
- oxygen_level: кислород
- metabolic_rate: скорость потребления

**Динамика:**
```python
# При движении (интенсивное потребление O2)
if movement_intensity > 0.5:
    glucose_level -= movement_intensity * 0.002 * dt
    oxygen_consumed += movement_intensity * 0.05 * dt
    
    if oxygen < demand:
        lactate_level += 0.01 * dt  # Переход на анаэробный метаболизм
else:
    # В покое - медленное восстановление
    glucose_level += 0.0005 * dt
    lactate_level *= 0.99  # Медленное выведение
```

**Влияние на мозг:**
- glucose_level используется для питания нейронов
- Низкий glucose → нейроны менее ответственны
- Высокий lactate → затруднена нейронная передача

**Связь с энергией:**
```
glucose_level → передается в глию как energy_level
glucose_level → модулирует возбудимость ретины (0.5-1.0 фактор)
```

### 2.7 ПЛАСТИЧНОСТЬ (FunctionalPlasticityManager & StructuralPlasticityManager)

**Назначение:** Симулировать обучение (детство) и адаптацию

**ФУНКЦИОНАЛЬНАЯ ПЛАСТИЧНОСТЬ (изменение веса синапсов):**
```python
# Hebbian learning: если нейроны активны вместе → синапс усиливается
if presynaptic_activity > 0 and postsynaptic_activity > 0:
    weight_change = presynaptic_activity * postsynaptic_activity * learning_rate
    
    # С наградой (дофамин) обучение быстрее
    if reward > 0:
        weight_change *= 2.0  # Двойное усиление
    
    weight += weight_change
else:
    # Без активности → медленное забывание
    weight *= 0.99
```

**СТРУКТУРНАЯ ПЛАСТИЧНОСТЬ (изменение связей):**
- В детстве (juvenile=True): новые синапсы могут формироваться
  - Probability = 0.001 за шаг если активны оба нейрона
  - новый synapse: weight случайный (0.1-0.9)
  
- Во взрослости: новые синапсы не образуются
  - Только существующие могут модифицироваться

**Энергетическое влияние:**
```python
# Обучение затратно энергетически
learning_energy_cost = weight_changes * 0.01  # Все изменения суммируются
glucose_level -= learning_energy_cost
```

### 2.8 МОТОРНАЯ ИЕРАРХИЯ (MotorHierarchy)

**Назначение:** Преобразовать моторные команды в физические действия

**Входы:**
- movement_command: [2] направление движения (x, y) в диапазоне [-1, 1]

**Выходы:**
- velocity: [2] скорость тела лягушки

**Процесс:**
```python
# Нормализовать команду
if norm(movement_command) > 0:
    direction = movement_command / norm
    magnitude = min(norm(movement_command), 1.0)
else:
    return [0, 0]

# Применить энергетическую модуляцию
energy_factor = max(0.3, game_energy_ratio)  # 30% скорости при нулевой энергии
magnitude *= energy_factor

# Добавить инерцию (плавное ускорение)
target_velocity = direction * magnitude * MAX_VELOCITY  # MAX_VELOCITY = 200
current_velocity = get_current_velocity()
acceleration = (target_velocity - current_velocity) * 0.1  # Коэффициент инерции
velocity = current_velocity + acceleration
```

### 2.9 ЯЗЫК (Tongue Extension & Retraction)

**Условия запуска:**
```python
if not tongue_extended and len(flies_visible) > 0:
    # Найти ближайшую видимую муху
    nearest_fly = min(flies_visible, key=lambda f: distance(f, frog))
    
    # Условие: энергии хватает должно быть
    if game_energy > 5.0:  # Нужно минимум 5 энергии для попытки
        extend_tongue(target=nearest_fly.position)
```

**Динамика языка:**
```python
# Язык удлиняется со скоростью 150 пикселей/секунду
if tongue_extended:
    tongue_length += 150 * dt
    
    # Язык максимально 150 пиксели (то есть 1 сек удлинения)
    if tongue_length >= 150:
        retract_tongue()

# Энергетическая стоимость активного использования языка
tongue_energy_cost = 0.2 * dt  # 0.2 энергии в секунду пока язык выдвинут
game_energy -= tongue_energy_cost
```

**Поимка:**
```python
# Проверить не касается ли конец языка какую-нибудь муху
tongue_tip = frog_position + direction * tongue_length

for fly in flies:
    if distance(tongue_tip, fly) < hit_radius:
        # Энергетическая модуляция успеха (низкая энергия = худше охота)
        energy_success_modifier = max(0.3, game_energy_ratio)
        success_prob = base_success_prob * energy_success_modifier
        
        if random() < success_prob:
            attached_fly = fly
            caught_flies += 1
            
            # НАГРАДА
            game_energy += 5.0
            game_energy = min(30.0, game_energy)
            
            dopamine_level += 0.3  # Дофаминовый всплеск
```

## 3. ЭНЕРГЕТИЧЕСКИЕ СИСТЕМЫ

### 3.1 СИСТЕМА 1: БИОЛОГИЧЕСКАЯ ЭНЕРГИЯ (0-1)

**Источник:** glucose_level в SystemicMetabolism

**Динамика:**
```python
# Расход при движении:
if movement_intensity > 0:
    glucose -= movement_intensity * neural_activity * 0.0001 * dt

# Расход при когнитивной работе (обучение):
glucose -= structural_plasticity_cost * dt

# Восстановление в покое:
if movement_intensity < 0.1:
    glucose += 0.0005 * dt  # Медленное восстановление

# Диапазон: [0.0, 1.0]
glucose = max(0.0, min(1.0, glucose))
```

**Влияние:**
- На глию: energy_level параметр для GlialNetwork.update()
- На нейромодуляцию: модулирует дофамин и серотонин
- На возбудимость: сенсорный вход ослабляется при низкой энергии

### 3.2 СИСТЕМА 2: ИГРОВАЯ ЭНЕРГИЯ (0-30)

**Инициализация:** self.game_energy = 30.0

**Динамика:**
```python
# Затрата на движение (как в ANN/SNN)
movement_cost = (0.08 + 0.02 * velocity_norm) * dt
game_energy -= movement_cost

# Награда за поимку
if caught_fly:
    game_energy += 5.0
    game_energy = min(30.0, game_energy)

# БЕЗ пассивного восстановления (в отличие от биологической)
# БЕЗ затрат на другое (только на движение)

game_energy = max(0.0, game_energy)
```

**Влияние:**
- На сенсорное восприятие: 
  ```python
  game_energy_ratio = game_energy / 30.0
  energy_excitability_factor = 0.5 + 0.5 * game_energy_ratio
  retinal_output *= energy_excitability_factor
  ```
- На успех охоты: 
  ```python
  success_prob *= max(0.3, game_energy_ratio)
  ```
- На скорость движения: 
  ```python
  velocity *= max(0.3, game_energy_ratio)
  ```

**Game Over условие:**
```python
if game_energy <= 0:
    print("GAME OVER: Energy depleted!")
    stop_simulation()
```

## 4. СИГНАЛЫ И ВЗАИМОДЕЙСТВИЯ

### 4.1 Полный граф сигналов

```
ОКРУЖЕНИЕ (мухи)
    ↓
СЕНСОРНАЯ СИСТЕМА (ретина)
    ├─→ retinal_output [16]
    └─→ (модулирована game_energy_ratio)
    
ТЕКТУМ
    ├─→ tectal_output [16]
    └─→ movement_command [2]
    
НЕЙРОМОДУЛЯЦИЯ
    ├─→ dopamine_level (0.5-0.85)
    ├─→ serotonin_level (0.5-0.75)
    └─→ (модулированы энергией)
    
ГЛИЯ
    ├─→ читает: tectal_activity, energy_level
    ├─→ выпускает: gliotransmitter
    └─→ модулирует: синапсы

ПЛАСТИЧНОСТЬ
    ├─→ читает: дофамин, активность нейронов
    ├─→ изменяет: веса синапсов
    └─→ стоит энергии

МЕТАБОЛИЗМ
    ├─→ glucose_level (источник биологической энергии)
    └─→ связан: с двигательной активностью

ИГРОВАЯ ЭНЕРГИЯ
    ├─→ game_energy (0-30)
    ├─→ затрачивается: на движение
    ├─→ восполняется: поимкой мух
    └─→ модулирует: все поведение
    
ЯЗЫК & ОХОТА
    ├─→ затрачивает: game_energy
    ├─→ генерирует: дофамин при награде
    └─→ модулируется: энергией (успех охоты)
    
МОТОРНАЯ ИЕРАРХИЯ
    ├─→ читает: movement_command, game_energy_ratio
    └─→ выдает: velocity [2] → физика

ФИЗИКА (Pymunk)
    └─→ двигает лягушку по сцене
```

## 5. ЖИЗНЕННЫЙ ЦИКЛ: ДЕТСТВО → ВЗРОСЛОСТЬ

### 5.1 ДЕТСТВО (0-5000 шагов, ≈50 сек)

**Характеристики:**
- self.brain.is_juvenile = True
- juvenile_duration = 5000 шагов

**Нейрохимия:**
```python
dopamine_level = 0.85  # Высокое - максимум исследования!
serotonin_level = 0.75  # Высокое - хорошее настроение
```

**Эффекты:**
- Нейроны более возбудимы
-学習율повышена (обучение быстрее)
- Синапсы быстро укрепляются и ослабляются
- Новые синапсы могут образоваться

**Пластичность:**
- Structural plasticity активна (новые связи)
- Functional plasticity быстрая (изменение весов × 2)
- Память формируется быстро

**Поведение:**
- Более случайное (включен "exploration noise")
- Риск выше (пробует неизвестные стратегии)
- Результаты в охоте нестабильны (но учится)

**Энергия:**
- Игровая энергия: 30.0 начально
- Биологическая энергия: 1.0 начально
- Расход: стандартный

### 5.2 ПЕРЕХОД (шаг 5000)

```python
if self.juvenile_age >= self.juvenile_duration:
    self.is_juvenile = False
    dopamine_level = 0.5  # Нормальный
    serotonin_level = 0.5  # Нормальный
    # Все синапсы "заморожены" в их текущих весах
```

### 5.3 ВЗРОСЛОСТЬ (5000+ шагов)

**Характеристики:**
- self.brain.is_juvenile = False
- juvenile_age продолжает считаться (только для логирования)

**Нейрохимия:**
```python
dopamine_level = 0.5  # Нормальный
serotonin_level = 0.5
# Модуляция энергией сохраняется!
```

**Эффекты:**
- Нейроны менее возбудимы (более консервативны)
- Обучение медленнее (структурная пластичность отключена)
- Синапсы стабилизированы
- Новые синапсы НЕ образуются

**Поведение:**
- Применение теми что выучилось в детстве
- Более стабильное (меньше случайностей)
- Результаты охоты стабильнее (меньше вариативности)
- Метаболизм оптимизированный

**Энергия:**

## 6. ФОРМУЛЫ И ПОРОГИ

### 6.1 Визуальная система

```python
# Кодирование направления в тектум (16 каналов)
angle_to_fly = atan2(fly.y - frog.y, fly.x - frog.x)
angle_normalized = (angle_to_fly + π) / (2π)  # [0, 1]
channel = int(angle_normalized * 16) % 16

# Интенсивность сигнала (зависит от расстояния)
distance = distance(fly, frog)
brightness = max(0, 1 - distance / visual_range)  # [0, 1]

# Энергетическая модуляция (КРИТИЧНО!)
retinal_signal = brightness * (0.5 + 0.5 * game_energy_ratio)  # [0, 1]

# Шум добавляется для реалистичности
retinal_signal += gaussian_noise(μ=0, σ=0.02)
retinal_signal = max(0, min(1, retinal_signal))
```

### 6.2 Тектальная обработка

```python
# Каждый канал тектума содержит 50 нейронов
# Каждый нейрон имеет threshold для возбуждения
neuron_threshold = 0.3  # 30% от максимального входа

# Активация нейрона
if retinal_input > neuron_threshold:
    neuron_activity = (retinal_input - threshold) / (1 - threshold)
else:
    neuron_activity = 0

# Tectal output - средняя активность по каждому каналу
tectal_output[i] = mean(neuron_activities_in_channel_i)
```

### 6.3 Энергетическая модуляция везде

```python
# СЕНСОРИКА
retinal_output *= (0.5 + 0.5 * game_energy_ratio)

# МОТОРИКА
velocity *= max(0.3, game_energy_ratio)

# ОХОТА
success_prob *= max(0.3, game_energy_ratio)

# НЕЙРОМОДУЛЯЦИЯ
dopamine *= max(0.3, energy_level)

# ГЛИЯ
if metabolic_demand > energy_cost_factor * 1.0:
    gliotransmitter *= 0.3  # Дефицит!
```

### 6.4 Затраты энергии

## 7. ГЛАВНЫЕ АРХИТЕКТУРНЫЕ ПРИНЦИПЫ

### 7.1 Энергия повсюду

❌ **НЕПРАВИЛЬНО:** Энергия влияет только на скорость

✅ **ПРАВИЛЬНО:** Энергия модулирует:
- Сенсорное восприятие (что мозг видит)
- Успех охоты (вероятность поймать)
- Скорость движения (как быстро двигаться)
- Нейромодуляцию (дофамин/серотонин)
- Глиальную работу (метаболическую поддержку мозга)

### 7.2 Двухуровневая обработка

У BioFrog ДОЛЖНЫ быть две энергии:
1. **Биологическая (0-1)** - для реалистичной динамики тела
2. **Игровая (0-30)** - для совместимости с ANN/SNN

### 7.3 Глия - главный модулятор

❌ **НЕПРАВИЛЬНО:** Глия просто выделяет вещества

✅ **ПРАВИЛЬНО:** Глия:
- Считывает энергию
- Проверяет может ли мозг позволить себе текущий уровень активности
- Подавляет синапсы если энергии не хватает
- Усиливает синапсы если есть избыток

### 7.4 Информационные потоки

```
energy → сенсорика → что видит мозг
energy → моторика → как быстро двигаться  
energy → нейромодуляция → настроение мозга
energy → глия → поддержка мозга
energy → охота → вероятность поимки
```

Всё должно быть СВЯЗАНО через энергию!

## ЗАКЛЮЧЕНИЕ

Вот полная архитектура BioFrog. Ключевые моменты для имплементации:

1. **Двойная энергия** - обе системы работают параллельно и независимо
2. **Энергия везде** - от сенсорики до охоты, всё модулируется
3. **Глия критична** - без неё энергия не влияет на мозг правильно  
4. **Информационные потоки** - мозг ДОЛЖЕН видеть energy_ratio как параметр
5. **Жизненный цикл** - детство и взрослость РЕЗКО отличаются

При имплементация с нуля главное:
- Сначала структура и сигналы
- Потом энергия везде (не забыть ни одного места)
- Потом тестирование что энергия влияет на каждый компонент
